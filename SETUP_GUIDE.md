# คู่มือการตั้งค่าระบบ (System Integration & Setup Guide)

คู่มือนี้จะอธิบายขั้นตอนการเชื่อมโยงระบบเว็บแอปพลิเคชันของคุณกับ **Firebase Firestore**, **Google Sheets API** และวิธีการใช้งาน**เครื่องมือนำเข้าข้อมูล Excel แบบกลุ่ม (Excel Bulk Importer)** ทั้งผ่านคำสั่ง CLI และหน้าต่างควบคุมบนหน้าเว็บ

---

## 1. การตั้งค่า Firebase & Firestore (Firebase Setup)

### ขั้นตอนการสร้างโครงการและฐานข้อมูล:
1. เข้าไปที่หน้า [Firebase Console](https://console.firebase.google.com/) แล้วเข้าสู่ระบบด้วยบัญชี Google ของคุณ
2. คลิก **Add Project** (เพิ่มโครงการ) และตั้งชื่อโครงการ เช่น `remix-clear-advance`
3. เปิดใช้งาน Google Analytics (ทางเลือก) จากนั้นกดสร้างโครงการสำเร็จ
4. ไปที่เมนู **Build > Firestore Database** แล้วคลิก **Create Database** (สร้างฐานข้อมูล)
5. **การกำหนดสิทธิ์การเข้าถึง (Security Rules):**
   - เพื่อความปลอดภัย แนะนำให้กำหนดกฎการเข้าถึงขั้นเริ่มต้นในไฟล์ `firestore.rules` ดังนี้:
     ```javascript
     rules_version = '2';
     service cloud.firestore {
       match /databases/{database}/documents {
         match /{document=**} {
           allow read, write: if true; // สามารถปรับแต่งสิทธิ์ผู้ใช้งานให้ปลอดภัยยิ่งขึ้นภายหลัง
         }
       }
     }
     ```

## 2. การตั้งค่า Service Account & Google Sheets API

ระบบนี้สามารถซิงค์ข้อมูลทั้งหมดจาก Firestore ไปยัง Google Sheets ได้โดยอัตโนมัติ เพื่อรักษาความเป็นระเบียบเรียบร้อยของข้อมูลตามโครงสร้าง 7 ตารางหลัก

### ขั้นตอนการขอสิทธิ์และการเปิดใช้งาน API:
1. เข้าไปที่ [Google Cloud Console](https://console.cloud.google.com/) เลือกโปรเจ็กต์ Firebase ของคุณ
2. ไปที่เมนู **APIs & Services > Library** จากนั้นค้นหาและกดเปิดใช้งาน (Enable):
   - **Google Sheets API**
   - **Google Drive API**
3. ไปที่เมนู **IAM & Admin > Service Accounts** แล้วคลิก **Create Service Account**:
   - ตั้งชื่อบัญชี เช่น `sheets-sync-worker`
   - มอบบทบาท (Role) เป็น **Editor** หรือ **Owner** ของโปรเจ็กต์
4. เมื่อสร้างเสร็จแล้ว คลิกเข้าที่ชื่อบัญชีบริการนั้น ไปที่แท็บ **Keys** (คีย์) > **Add Key** > **Create new key** เลือกรูปแบบ **JSON**
5. ระบบจะดาวน์โหลดไฟล์คีย์ JSON มาเก็บไว้ที่เครื่องคอมพิวเตอร์ของคุณ (ไฟล์นี้ห้ามเผยแพร่เด็ดขาด)
6. นำข้อมูลในไฟล์ JSON ไปกำหนดค่าลงในไฟล์ `.env` ของโครงการเพื่อใช้งานเชื่อมต่อ

### ⚠️ สิ่งสำคัญที่สุด (มักถูกลืมบ่อย):
คุณจำเป็นต้องสร้าง Google Spreadsheet ขึ้นมาใหม่ จากนั้น**กดปุ่ม Share (แชร์) และระบุอีเมลของ Service Account** (ที่ปรากฏอยู่ในไฟล์ JSON คีย์ ช่อง `client_email` ตัวอย่างเช่น `sheets-sync-worker@xxxx.iam.gserviceaccount.com`) ให้มีสิทธิ์เป็น **Editor** ของสเปรดชีตนั้น เพื่อให้ระบบหลังบ้านสามารถอ่านเขียนข้อมูลได้

## 3. วิธีการนำเข้าไฟล์ Excel เข้าสู่ Firestore (Excel Bulk Importer)

ระบบถูกออกแบบมาให้รองรับการอัปโหลดนำเข้าตารางข้อมูลสำคัญ 7 ตารางหลัก โดยจะรักษาข้อมูลดั้งเดิมทุกคอลัมน์เก็บไว้ในฟิลด์ `rawData` และเพิ่มฟิลด์ตรวจสอบย้อนกลับ (`sourceFile`, `sourceSheet`, `sourceRow`, `importedAt`) พร้อมระบบลบข้อผิดพลาดเมื่อมีข้อมูลซ้ำ

### วิธีที่ 1: นำเข้าผ่านหน้าเว็บบอร์ด (แนะนำ - สะดวกและรวดเร็วที่สุด)
1. เข้าสู่ระบบเว็บแอปพลิเคชันด้วยบัญชีผู้มีสิทธิ์ระดับ **ผู้ดูแลระบบ (Admin)**
2. ไปที่เมนู **แผงผู้ดูแลระบบ (Admin Settings)**
3. คลิกแท็บ **"นำเข้า Excel เข้า Firestore"** (จะมีรูปไอคอนฐานข้อมูลสีส้ม)
4. ลากไฟล์ Excel (.xlsx) หรือคลิกเพื่ออัปโหลดไฟล์
5. ระบบจะทำการแยกวิเคราะห์หน้าชีท (Sheets) ทั้งหมดโดยอัตโนมัติ:
   - แสดงคอลัมน์และแถวข้อมูลทั้งหมดที่ตรวจพบ
   - ทำการวิเคราะห์และแนะนำ Collection ปลายทางให้อัตโนมัติ (เช่น ชีทที่มีรายชื่อจะแมตช์เข้า `employees`, ยอดเงินทดรองแมตช์เข้า `advances`)
   - ผู้ใช้สามารถกดเลือก Collection ปลายทางอื่นเพื่อปรับแต่งได้อย่างอิสระ
6. คลิกเพื่อเปิดตรวจสอบดูตารางตัวอย่าง (Preview 5 แถวแรก) ได้ทันที เพื่อความถูกต้อง
7. กดปุ่ม **"นำเข้าข้อมูลที่เลือกทั้งหมดเข้าสู่ Firestore"**
8. ระบบจะเขียนข้อมูลเข้าฐานข้อมูลแบบปลอดภัย (Upsert - เขียนทับกรณี ID ซ้ำกัน ป้องกันข้อมูลซ้อน) และทำการคำนวณสรุปบัญชีแยกประเภท (GL), ยอดสรุปต้นทุนโครงการ และเอกสารติดตามทันที

### วิธีที่ 2: รันนำเข้าผ่าน CLI (Command Line Interface)
หากต้องการประมวลผลไฟล์ขนาดใหญ่บนเครื่องเซิร์ฟเวอร์โดยตรง สามารถเปิดใช้งานสคริปต์ได้โดยทำตามขั้นตอนด้านล่าง:

1. นำไฟล์ Excel ไปวางไว้ที่โฟลเดอร์รากของโครงการ
2. เปิด Terminal และใช้คำสั่งรันสคริปต์:
   ```bash
   npx tsx import-excel.ts <ชื่อไฟล์ของคุณ>.xlsx
   ```
3. สคริปต์จะแสดงผลสถานะการทำงาน แยกระหว่างการประมวลผลแถว การกรองค่าคอลัมน์ ไปจนถึงสถานะบันทึกสมบูรณ์แบบเรียลไทม์

## 4. ระบบการซิงค์และประมวลผลอัตโนมัติ (Automated Workflows)

- **ระบบการคำนวณภายใน (Internal Real-time Compute):** ทุกครั้งที่คุณกดนำเข้าไฟล์ Excel ผ่านระบบเบราว์เซอร์ หรือเมื่อหน้าแดชบอร์ดหลักได้รับการโหลด ตัวประมวลผลจะวิ่งไปคำนวณและอัปเดต collections ตัววิเคราะห์ผลลัพธ์ของ Firestore ทั้งหมดแบบอัตโนมัติ (`projects`, `project_costs`, `document_tracking`, `GL`) โดยไม่ต้องกดทำด้วยมือ
- **ระบบการซิงค์เข้า Google Sheets:** เมื่อเปิดใช้ระบบซิงค์ Google Workspace ข้อมูลล่าสุดที่มีผลจากการวิเคราะห์และปรับปรุงจะถูกดึงส่งตรงไปยัง Google Sheets ทันทีอย่างถูกต้องแม่นยำ 100% ครบทั้ง 7 แผ่นงานย่อยย่อยตามที่คุณต้องการ
